Interpretable Context Methodology
Folder structure as agent architecture
ICM replaces framework-level orchestration with filesystem structure. Numbered stage folders represent workflow steps. Plain markdown files carry the context and instructions. One agent reads the right files at the right moment — no coordination framework required.

ICM Design Principles
How the framework is structured
Each of Van Clief's five ICM principles maps directly to design decisions across all three workspace types, plus one additional pattern we added.
01
One stage, one job
Each numbered folder handles exactly one step. A stage that decomposes never also scores. Enforced via STAGE.md scope.
02
Plain text interface
All inter-stage communication via markdown files. No database. No API calls between stages. Human-editable at any point.
03
Every output editable
Agent stops after each stage. Human opens output files, edits freely. Agent uses edited versions as authoritative.
04
Configure the factory
_core/ holds stable reference material. Set up once. Run many times.
05
Progressive disclosure
CLAUDE.md (~400 tok) always loaded. STAGE.md (~300 tok) swaps per stage. Skills load on demand, one at a time.
Our addition to ICM
Charter as constitutional memory
ICM's original design doesn't address bounded agents or scope enforcement. We add the Charter as a constitutional memory layer: always loaded first, overrides everything else, human-writable only. The agent can propose charter amendments but cannot self-approve them. This solves the scope creep problem directly in the filesystem — not in code.
Progressive disclosure — context window usage per layer
CLAUDE.md (always loaded)
~400 tok
Charter (always, ops workspace)
~300 tok
SESSION.md (each session)
~250 tok
STAGE.md (current stage only)
~300 tok
One skill (loaded on demand)
~1.5k tok
Stage input files (current stage)
~1k tok
Deep reference (rarely loaded)
5k+ tok
Context rot prevention: Loading all skills, all memory, and all stage outputs simultaneously causes the agent to blend contexts — producing outputs that violate scope or use the wrong skill's patterns. Progressive disclosure keeps each stage focused on exactly its single job.

ICM (Van Clief)
Folder-as-architecture, one stage one job, plain text, progressive disclosure		The organizing principle for all three workspaces.
Every CLAUDE.md, STAGE.md, and _core/ structure.		All three workspaces
The complete synthesis: ICM provides the structural pattern (how to organize files and stages). Our frameworks provide the content (what each stage does, what skills contain). The chartered agent pattern provides governance (what the agent can and cannot do). Progressive disclosure optimizes context (what gets loaded when). Together: a complete, deployable, bounded agent system.
